响应式数据和非响应式数据的区别

  1. 响应式数据：Vue中的响应式数据是指通过Vue实例的数据对象(data)或计算属性(computed)定义的数据。当响应式数据发生改变时，Vue会自动检测到数据的变化，并立即触发相关的UI更新。这意味着你不需要手动去更新UI，而是通过更新数据来响应用户操作或其他数据变化。
  2. 非响应式数据：非响应式数据是指在Vue实例之外定义的数据，或者在Vue实例创建后动态添加到实例上的数据。对于非响应式数据，当其发生改变时，Vue不会自动检测到变化，也不会触发UI的更新。你需要手动调用Vue的方法来更新UI，例如$forceUpdate或重新渲染组件。

v-model 绑定其他组件传递过来的数据

![image-20230708105019515](E:\Blogger\blog\source_posts\Vue2 常用标签\image-20230708105019515.png)

  • v-model 是双向数据绑定，所以修改任意一个方向的数据，都会改变，而当一个数据是其他组件传递过来的时`候，这个数据只能被数据的所有者修改，所以此时应该将v-model 拆成 底层实现， : value + @监听事件=”方法名”,通过子传父，传递给父组件修改

  • 简化版，以为v-model 底层实现是 value+@input，所以传递值时可以直接写 v-model=要传递的值，然后子组件就可以直接使用双向绑定，绑定事件中通过$emit(‘input’) 向父组件传值，事件的值在e.target.value 中获取

  • ![image-20230708112243841](E:\Blogger\blog\source_posts\Vue2 常用标签\image-20230708112243841.png)

  • 在传递的值后面加上.sync 子组件就可以直接进行双向数据绑定，将修改的数据传递给父组件时，固定写法是$emit(‘update:值的名字’,要传递的值)

  • ![image-20230708112218878](E:\Blogger\blog\source_posts\Vue2 常用标签\image-20230708112218878.png)

  • .sync 修饰符

  • ![image-20230710100209580](E:\Blogger\blog\source_posts\Vue2 常用标签\image-20230710100209580.png)

  • 直接在 子组件的引用处 加上 :属性名.sync=”父组件值”,然后子组件就可以直接 通过 props 加 属性名 拿到属性的值，可以直接使用双向绑定，要修改的时候写一个方法，通过 this.$emit('update:属性值',你要传的值)去修改值。 .sync的本质是 :属性值 + @update
