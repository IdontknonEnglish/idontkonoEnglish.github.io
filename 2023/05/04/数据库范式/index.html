数据库范式

关键码

  1. 超键：在关系中能唯一标识元组的属性或属性集称为关键模式的超键。(既至少包含一个候选键的属性集)

  2. 候选键：不含有多余属性的超键称为候选键。也就是在候选键中在删除属性就不是键了。(id等唯一且可以确定身份)

  3. 主键：用户选作元组标识的候选键称为主键。一般不加说明，键就是指主键。(用户选择的候选键)

  4. 外键：如果模式R中属性K是其他模式的主键，那么K在模式R中称为外键。(另一张表的主键，在这张表称为外键)

  5. 主属性: 所有候选键的集合

完全依赖、部分依赖、传递依赖

部分函数依赖：存在X→Y，若X’是X的真子集，存在X’→Y

完全函数依赖：存在X→Y，若X’是X的真子集，不存在X’→Y

传递函数依赖：因为 X -> Y, Y -> Z, 所以 X -> Z

范式

 1. 1NF (关系中的每一个元素都不可以再分，例如 一本书的作者有两个 ，需要将他们拆开分成 两列)
 2. 2NF (非主属性部分依赖于主属性, 既非候选键只依赖候选键中的一个属性, 这个时候就要拆表。 例如：歌曲名称和歌手可以组成候选键 经纪公司是非候选键 经纪公司 依赖于歌手 但是不依赖于 歌曲名称 )
 3. 3NF (表中的所有数据元素不但要能唯一的被主关键字所标识，而且他们之间还要相互独立，不存在其他的函数关系，既在2NF的基础上消除 传递函数依赖)
 4. BCNF 消除主属性对主键的部分依赖和传递依赖
 5. 4NF 消除表中的多值依赖(例如 所属专辑 可以确定发表时间 和 专辑类型 但是 发表时间 和 专辑类型 之间并没有联系 需要将他们拆表 分成 所属专辑，发表时间 和 所属专辑，专辑类型 两张表)
