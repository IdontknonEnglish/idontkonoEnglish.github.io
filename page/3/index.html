    分布式数据库系统的透明性：

    分片透明:

    ​ 是指用户不必关心数据是如何分片的，它们对数据的操作在全局关系上进行，即关心如何分片对用户是透明的，因此，当分片改变时应用程序可以不变。分片透明性是最高层次的透明性，如果用户能在全局关系一级操作，则数据如何分布，如何存储等细节自不必关心，其应用程序的编写与集中式数据库相同。

    复制透明:

    ​ 用户不用关心数据库在网络中各个节点的复制情况，被复制的数据的更新都由系统自动完成。在分布式数据库系统中，可以把一个场地的数据复制到其他场地存放，应用程序可以使用复制到本地的数据在本地完成分布式操作，避免通过网络传输数据，提高了系统的运行和查询效率。但是对于复制数据的更新操作，就要涉及到对所有复制数据的更新。

    位置透明:

    ​ 是指用户不必知道所操作的数据放在何处，即数据分配到哪个或哪些站点存储对用户是透明的。

    局部映像透明性（逻辑透明)

    ​ 是最低层次的透明性，该透明性提供数据到局部数据库的映像，即用户不必关心局部DBMS支持哪种数据模型、使用哪种数据操纵语言，数据模型和操纵语言的转换是由系统完成的。因此，局部映像透明性对异构型和同构异质的分布式数据库系统是非常重要的。

    笛卡尔积: 符号是 （×）

    自然连接 符号是（⋈）

    ​ 在R表和S表中 找到相同属性，值相同的元组，把他们连接在一起，相同的属性只出现一次， 去重

  16. 什么是二叉树？什么是有序二叉树？

    二叉树（binary tree）是指树中节点的度不大于2的有序树，它是一种最简单且最重要的树。二叉树的递归定义为：二叉树是一棵空树，或者是一棵由一个根节点和两棵互不相交的，分别称作根的左子树和右子树组成的非空树；左子树和右子树又同样都是二叉树 。

    有序二叉树，就是左子树上的数值小于树根上的值，树根的值小于右子树的值。

数据库范式

关键码

  1. 超键：在关系中能唯一标识元组的属性或属性集称为关键模式的超键。(既至少包含一个候选键的属性集)

  2. 候选键：不含有多余属性的超键称为候选键。也就是在候选键中在删除属性就不是键了。(id等唯一且可以确定身份)

  3. 主键：用户选作元组标识的候选键称为主键。一般不加说明，键就是指主键。(用户选择的候选键)

  4. 外键：如果模式R中属性K是其他模式的主键，那么K在模式R中称为外键。(另一张表的主键，在这张表称为外键)

  5. 主属性: 所有候选键的集合

完全依赖、部分依赖、传递依赖

部分函数依赖：存在X→Y，若X’是X的真子集，存在X’→Y

完全函数依赖：存在X→Y，若X’是X的真子集，不存在X’→Y

传递函数依赖：因为 X -> Y, Y -> Z, 所以 X -> Z

范式

 1. 1NF (关系中的每一个元素都不可以再分，例如 一本书的作者有两个 ，需要将他们拆开分成 两列)
 2. 2NF (非主属性部分依赖于主属性, 既非候选键只依赖候选键中的一个属性, 这个时候就要拆表。 例如：歌曲名称和歌手可以组成候选键 经纪公司是非候选键 经纪公司 依赖于歌手 但是不依赖于 歌曲名称 )
 3. 3NF (表中的所有数据元素不但要能唯一的被主关键字所标识，而且他们之间还要相互独立，不存在其他的函数关系，既在2NF的基础上消除 传递函数依赖)
 4. BCNF 消除主属性对主键的部分依赖和传递依赖
 5. 4NF 消除表中的多值依赖(例如 所属专辑 可以确定发表时间 和 专辑类型 但是 发表时间 和 专辑类型 之间并没有联系 需要将他们拆表 分成 所属专辑，发表时间 和 所属专辑，专辑类型 两张表)

一、获取Stream流

  1. 根据类型不同获取方法有4中

    1. 单列集合如 List 和 Set 可以直接通过 .stream()获取

    2. 双列集合如 Map 需要先转化为单例再获取 如 map.keySet().stream()

    3. 数组 直接通过工具方法获取 如 Arrays.stream(数组名)

    4. 零散数据如 {1,2,3,4,5,6} , 可以通过Stream获取 例如 Stream.of({1,2,3,4,5,6})

              1. 如果是引用类型例如String 可以将其先转化为数组再进行第三步操作
